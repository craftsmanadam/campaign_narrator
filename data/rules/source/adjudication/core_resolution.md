# Core Resolution

The SRD resolves uncertain outcomes with D20 Tests. There are three kinds:

- ability checks,
- saving throws,
- attack rolls.

## D20 Test Procedure

1. Roll `1d20`.
2. Apply Advantage or Disadvantage if relevant.
3. Add the relevant ability modifier.
4. Add Proficiency Bonus if the creature is proficient in the thing being used.
5. Apply circumstantial bonuses or penalties.
6. Compare the total to the target number.

## Target Numbers

- Ability checks and saving throws use a Difficulty Class (`DC`).
- Attack rolls target Armor Class (`AC`).
- Success occurs when the total equals or exceeds the target number.

## General Before Exceptions

The SRD follows a general-rule-first model. Class features, feats, spells, magic items, and monster abilities can override general rules. When a specific exception conflicts with a general rule, the exception wins.

## Important Prototype Principle

The prototype should prefer explicit rule exceptions over inferred general behavior. If a feature says it changes the normal resolution procedure, the feature should take priority.
